# 開発を進めるために必要な資料チェックリスト

このドキュメントは、`docs/実現計画.md` を実行に移すために必要な資料を、
**優先度順に整理した実務用チェックリスト**です。

## 0. 使い方
- 優先度は `Must（着手前必須） / Should（初期実装中に整備） / Could（MVP後でも可）`。
- `Status` は `未着手 / 作成中 / 完了` を入れる。
- 迷ったら Must から順に作る。

---

## 4. すぐ使えるテンプレート（最小）

## 4.1 要件定義書テンプレ（MVP）
1. 背景・目的
2. 対象ユーザー
3. ユースケース
4. 機能要件（必須/任意）
5. 非機能要件（起動速度、通知遅延許容など）
6. 非スコープ
7. 受け入れ条件

## 4.2 通知仕様書テンプレ
1. 通知トリガー条件
2. 配信時間帯・頻度
3. ユーザー設定可否
4. 通知文面ルール
5. タップ時遷移
6. 権限拒否時の導線
7. OS制約時のフォールバック

## 4.3 データ仕様テンプレ（CSV）
- カラム名
- 型
- 必須/任意
- バリデーション規則
- サンプル値
- NG例
